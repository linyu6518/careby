AI 助手集成计划 - ChatGPT API

一、功能需求分析

1.1 核心功能

  • 智能问答：回答基础问题（FAQ相关）
  • 页面导航：跳转到服务页面、会员页面等
  • 表单操作：打开预约表单、咨询表单
  • 联系信息：提供电话、邮箱等联系方式
  • 服务推荐：根据用户问题智能推荐相关服务

1.2 UI/UX 设计

  • 桌面端：右下角悬浮按钮（固定位置）
  • 移动端：与现有 CTA 按钮并列显示（底部左侧和右侧）
  • 聊天窗口：可展开/收起的对话界面
  • 多语言支持：支持英语、简体中文、繁体中文

二、技术架构

2.1 组件结构

src/

  hooks/
    └── useChatbot.ts            # Chatbot 状态管理 Hook

2.2 API 集成方案

方案：使用环境变量 + Vercel Serverless Functions

由于直接在前端调用 ChatGPT API 会暴露 API Key，采用以下架构：

前端 (React) 
  ↓
Vercel Serverless Function (/api/chat)
  ↓
OpenAI ChatGPT API

优势：

  • API Key 安全存储在服务器端
  • 可以添加速率限制和错误处理
  • 支持流式响应（Streaming）
  • 易于监控和日志记录

2.3 数据流

graph TD
    User[用户输入] --> ChatWindow[聊天窗口]
    ChatWindow --> useChatbot[useChatbot Hook]
    useChatbot --> APICall[调用 /api/chat]
    APICall --> Serverless[Vercel Serverless Function]
    Serverless --> ChatGPT[OpenAI ChatGPT API]
    ChatGPT --> Serverless
    Serverless --> Streaming[流式响应]
    Streaming --> ChatWindow
    ChatWindow --> ActionHandler[操作处理器]
    ActionHandler --> Navigation[页面导航]
    ActionHandler --> Form[打开表单]
    ActionHandler --> Contact[显示联系方式]

三、实现细节

3.1 悬浮按钮设计

桌面端：

  • 位置：fixed bottom-6 right-6
  • 样式：圆形按钮，带 AI 图标
  • 动画：悬停效果、点击波纹
  • z-index: z-[9998] (低于移动端按钮的 9999)

移动端：

  • 位置：fixed bottom-[30px] right-[30px]
  • 与现有按钮并列：左侧 CTA 按钮，右侧 AI 按钮
  • 尺寸：与 CTA 按钮协调
  • 响应式：sm:hidden 隐藏，移动端显示

3.2 聊天窗口设计

布局：

  • 宽度：桌面端 400px，移动端全屏
  • 高度：桌面端 600px，移动端 80vh
  • 位置：从右下角滑入动画
  • 关闭：点击外部或关闭按钮

功能区域：

  1. 头部：标题、语言切换、关闭按钮
  2. 消息区：滚动消息列表，支持 Markdown 渲染
  3. 输入区：文本输入、发送按钮、快捷操作
  4. 底部：常用问题快捷按钮

3.3 ChatGPT API 集成

提示词设计：

  • 系统提示：定义助手角色、可用功能、响应格式
  • 上下文管理：保留最近 10 轮对话
  • 功能调用：识别用户意图（导航、表单、联系等）

响应格式：

interface ChatResponse {
  message: string;           // AI 回复文本
  action?: {                 // 可选操作
    type: 'navigate' | 'form' | 'contact' | 'service';
    data: any;
  };
  suggestions?: string[];     // 推荐问题
}

3.4 操作处理

页面导航：

  • 识别关键词：“服务”、“会员”、“FAQ”、“关于我们”
  • 使用 React Router 或 hash 导航
  • 平滑滚动到目标区域

表单操作：

  • 打开预约表单模态框
  • 预填充相关信息（如服务类型）
  • 触发现有表单组件

联系信息：

  • 显示电话号码（可点击拨打）
  • 显示邮箱（可点击发送邮件）
  • 显示地址（可打开地图）

服务推荐：

  • 分析用户问题关键词
  • 匹配相关服务（护理、远程医疗、疼痛管理等）
  • 提供服务卡片和链接

八、安全考虑

8.1 API Key 保护

  • ✅ 存储在 Vercel 环境变量中
  • ✅ 仅在 Serverless Function 中使用
  • ✅ 不在前端代码中暴露

8.2 速率限制

  • 每个用户每分钟最多 10 次请求
  • 使用 IP 地址或用户标识符限制
  • 防止滥用和成本控制

8.3 内容过滤

  • 过滤不当内容
  • 限制对话主题范围（仅限 Careby 相关）
  • 错误处理和降级方案

十三、成本估算

13.1 OpenAI API 成本

  • GPT-4 Turbo: ~$0.01 per 1K tokens
  • 平均对话：~500 tokens per request
  • 预计月成本：$50-200（取决于使用量）

13.2 优化建议

  • 使用 GPT-3.5 Turbo 降低成本（~$0.0005 per 1K tokens）
  • 实现缓存机制
  • 设置使用量限制
